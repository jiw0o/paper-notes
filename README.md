# Paper Notes

논문과 학습 주제를 정리하고 서로 연결하는 개인용 정적 웹앱입니다. 별도 서버나 데이터베이스 없이 GitHub Pages에서 실행됩니다.

## 기능

- 연구 현황, 최근 기록, 카테고리를 보여주는 메인 대시보드
- 논문 추가, 수정, 삭제 및 연도/카테고리/키워드/출간처 관리
- 제목, 저자, 태그 검색과 필터/정렬
- 논문마다 독립된 상세 페이지와 읽기 노트 자동 저장
- 제목, 소제목, 목록, 강조를 지원하는 문서형 노트와 기본 템플릿
- `[[논문 제목]]` 문법으로 다른 논문 언급 및 백링크 생성
- 즐겨찾기, 그리드/목록 보기
- 전체 데이터를 JSON으로 백업하고 복원
- 모바일 반응형 화면

## 로컬 실행

파일을 직접 열어도 되지만, 로컬 서버를 사용하면 브라우저 동작이 더 일관적입니다.

```bash
cd paper-notes
python3 -m http.server 8080
```

브라우저에서 `http://localhost:8080`을 엽니다.

## GitHub Pages 배포

1. 이 폴더를 새 GitHub 저장소에 push합니다.
2. 저장소의 **Settings → Pages**로 이동합니다.
3. **Build and deployment**의 Source를 **Deploy from a branch**로 설정합니다.
4. `main` 브랜치와 `/ (root)`를 선택하고 저장합니다.

잠시 후 `https://<사용자명>.github.io/<저장소명>/`에서 열립니다.

## 데이터에 관해 꼭 알아둘 점

데이터는 브라우저의 `localStorage`에 저장됩니다. 따라서 같은 사이트 주소를 같은 브라우저에서 열 때만 자동으로 유지됩니다. 기기 간 동기화는 되지 않으며 브라우저 데이터를 지우면 사라질 수 있으므로, 사이드바의 **데이터 내보내기**로 주기적으로 JSON 백업을 보관하세요.

GitHub Pages에 공개 저장소로 배포하더라도 브라우저에서 작성한 노트가 저장소에 자동 업로드되지는 않습니다. 다만 사이트 자체는 공개되므로 개인 전용 인증이 필요하다면 별도 인증/백엔드 구성이 필요합니다.
